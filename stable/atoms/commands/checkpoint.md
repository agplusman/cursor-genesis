# Command: /checkpoint

保存当前任务状态，便于后续恢复。

## 触发场景

- 长任务进行中，手动保存进度
- 即将切换话题或关闭对话前
- 完成一个阶段后，显式标记检查点

## 执行步骤

1. 在项目根目录创建或更新 `NOTES.md`
2. 记录以下信息：
   - 当前任务描述
   - 已完成的步骤
   - 下一步计划
   - 关键文件列表
   - 时间戳
3. 在 `Checkpoint 历史` 区域追加一条记录
4. 确认用户："已保存 checkpoint，可通过 /resume 恢复"

## NOTES.md 结构

```markdown
# 工作笔记

## 当前任务
[任务描述]

## 进度
- [x] 步骤 1
- [x] 步骤 2
- [ ] 步骤 3（下一步）

## 关键文件
- path/to/file1
- path/to/file2

## Checkpoint 历史
- [时间] 完成步骤 2，下一步：步骤 3
```
