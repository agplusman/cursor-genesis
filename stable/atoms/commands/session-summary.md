# Command: Session Summary（会话沉淀）

## Goal

基于当前 Chat 的完整上下文，输出一份**描述性沉淀记录**，用于复盘或向他人展示工作过程。

## 执行方式

直接根据下方模板，基于本对话中已发生的内容，填充并输出结构化总结。无需额外读取文件，当前对话上下文即输入。

## 输出模板

```markdown
# 会话沉淀 | {简短主题描述}

> 生成时间：{YYYY-MM-DD}
> 原始对话：本 Chat 会话

## 一、用户初始问题

{用户最开始提出的问题或需求，用 1～3 句话概括}

## 二、思考方向

{对话中探索的主要思路、尝试的方向、关键决策点}

## 三、解决方案

{最终采用的方案或结论，以及主要产出}

## 四、发现的问题

{过程中遇到的障碍、意外发现、需要澄清的点}

## 五、解决过程

{每个问题是如何被解决的，或当前状态（待解决/部分解决）}

## 六、遗留与后续

{如有未完成事项、待观察点、建议的下一步}
```

## 约束

- 只记录**本对话中实际发生**的内容，不臆测或 extrapolate
- 若某节无对应内容，写「（无）」或简短说明
- 输出为 Markdown

## 保存约定（用户操作记录统一存放）

若用户要求保存到文件，默认路径：

- **有 `meta/user/`**：`meta/user/session-summaries/{YYYY-MM-DD}-{主题slug}.md`
- **无 `meta/user/`**：由用户指定或项目根目录

用户也可复制到任意位置。
