# Agent 自主性选择指南

> 回答：什么任务应该让 AI 自由发挥，什么任务需要严格控制？

## 自主性连续谱

Cursor 2026 形成了从最低到最高自主性的完整连续谱：

```
Tab 补全 → Inline Edit → Agent Chat → Plan Mode → Background Agent → Long-running Agent → Cloud Agent
最低自主性                                                                              最高自主性
```

## 匹配表

| 自主性级别 | 适合场景 | 人的角色 | 风险 |
|:---|:---|:---|:---|
| **Tab/Inline** | 小函数、已知模式、格式化 | 逐字符/逐行控制 | 极低 |
| **Agent Chat** | 单文件修改、明确需求、Debug | 对话式指导 | 低 |
| **Plan Mode** | 大功能、重构、跨文件修改 | 审批计划后放手 | 中 |
| **Background** | 文档生成、测试编写、依赖升级 | 只审查结果 | 中高 |
| **Long-running** | 复杂多文件特性、大型 PR | 定义目标后离开 | 高 |
| **Cloud Agent** | 独立任务、CI/CD 集成 | 异步审查 | 高 |

## 选择原则

### 1. 自主性越高 ≠ 越好

- 自主性越高，**人的认知负担越低**（不用实时盯着）
- 但**审查负担越高**（结果可能大量偏离预期）
- 而且**偏离风险越高**（AI 在长时间运行中可能走错方向）

### 2. 价值判断只能由人做

无论自主性多高，以下决策**不能委托给 AI**：
- 这个功能是否值得做？（价值判断）
- 两个方案选哪个？（方向选择）
- 什么时候发布？（风险评估）

### 3. 匹配任务特征，不匹配个人偏好

| 任务特征 | 推荐自主性 |
|:---|:---|
| 输出高度确定，有标准答案 | 低（Tab/Inline） |
| 需要创造性探索 | 中（Agent Chat） |
| 范围大但方向明确 | 高（Plan Mode → Background） |
| 高风险、不可逆操作 | 低（手动控制） |

## 实用技巧

1. **确保 Rules 和 Skills 完备再提升自主性**：Background/Cloud Agent 依赖预配置的上下文，没有 Rules 引导的高自主性 = 脱缰野马
2. **先 Plan 后 Execute**：在提升自主性之前，用 Plan Mode 让 AI 出计划，审批后再放手
3. **小步验证**：长时任务设置检查点（checkpoint），定期审查中间产物
4. **错误成本决定自主性上限**：线上环境用低自主性，开发环境可以放开
