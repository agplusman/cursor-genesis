# PPT 案例提取技巧

> 创建日期：2026-02-25
> 来源：AI 协作研发 PPT 制作过程

---

## 核心技巧

在整理应用场景案例时：

1. **追问细节亮点**：跟用户深入讨论，挖掘具体的协作细节和亮点
2. **主动搜索项目**：从相关项目代码中搜索，发现用户可能遗漏的亮点
3. **分类归纳**：把所有亮点分类，每类选一个典型展示
4. **灵活总结**：
   - 如果一个场景下有多个分类，可以每类一个小总结
   - 也可以所有分类统一做一个大总结
   - 根据内容复杂度和表达需要灵活选择

## 应用示例：AI 协作研发 PPT

### 场景划分

| 场景 | 核心展示 | 工具选择 |
|:---|:---|:---|
| 新功能开发 | 从 0 到 1 的完整流程 | Cursor |
| 旧项目改造 | Human in the Loop 的不同深度 | Cursor |
| 独立任务 | 批量处理、提示词复用 | Claude Code / Cursor |

### 新功能开发的子场景（以风险事件模块为例）

1. **项目理解**：AI 扫描项目，生成 DDD 文档，理解已有模块（组织机构、人员、风险等）
2. **需求开发**：描述每页展示内容，AI 生成前后端代码
3. **测试验证**：AI 辅助编写测试用例
4. **权限集成**：数据权限注解，需要全局视角
5. **导出功能**：报表导出，涉及格式处理

### 旧项目改造的子场景

1. **结构驱动**（饼图）：人定结构和交互，AI 定技术实现
2. **需求驱动**（多选组件）：需求明确，AI 一步到位
3. **探索驱动**（动态列宽）：AI 辅助分析 → 人产生灵感 → 迭代完善

## 总结

案例提取的本质是：**从散点到结构**。

- 散点：用户零散的记忆、项目中的代码片段
- 结构：分类、归纳、典型化

好的案例不是"多"，而是"有代表性"。

## 附：索引的重要性

在大型跨项目场景下，即使 Claude Code 支持 1M 上下文，也容易"找不到东西"。

**问题**：
- 项目多、文件散，AI 不知道从哪找
- 每次都要重新探索，效率低
- 相关内容分布在不同项目，难以关联

**解决**：建立索引层
- 主题索引：某个主题涉及哪些项目的哪些文件
- 节点注册：每个叶子节点暴露什么内容
- 关联维度：跨节点的知识关联

**本质**：索引是"找知识的路径"，让 AI 知道去哪找，而不是漫无目的地搜索。
