# 待整理区域设计

> 配套文档：03-知识增长机制.md 第二章「经验沉淀模式」
> 版本：v1.0
> 创建时间：2026-02-23

---

## 一、定位

「待整理」区域是经验沉淀模式的暂存区：
- 存放即时记录的粗糙内容
- 不要求结构完整
- 定期复盘后转入正式知识库

---

## 二、设计原则

### 2.1 低摩擦优先

> 记录的阻力越小，越容易���持

- 不强制格式，能记下来就行
- 支持多种输入方式（文件、语音转文字、截图）
- 分类可以后补

### 2.2 可追溯

- 记录时间戳
- 记录来源场景（哪个项目、什么事件）
- 方便复盘时回忆上下文

### 2.3 定期清理

- 不是永久存储
- 复盘后要么转正、要么删除
- 避免堆积成垃圾场

## 五、复盘流程

### 5.1 触发时机

| 触发 | 频率 | 范围 |
|------|------|------|
| 周复盘 | 每周日 | quick/ 目录 |
| 项目结束 | 项目完成时 | 对应 by-project/ 目录 |
| 月度复盘 | 每月末 | 全部 inbox/ |

### 5.2 复盘步骤

```
1. 浏览待整理内容
   ├─ 快速扫描，标记有价值的条目
   └─ 删除明显无价值的内容

2. 分类处理
   ├─ 值得沉淀 → 用四要素重写 → 进入02入库流程
   ├─ 需要补充 → 标记TODO，下次继续
   └─ 已过时/无价值 → 删除

3. 更新索引
   └─ 在 _index.md 中标记已处理项

4. 清理空文件
   └─ 删除已清空的日期文件/项目目录
```

### 5.3 复盘检查清单

```markdown
## 周复盘检查清单

- [ ] quick/ 目录本周记录已浏览
- [ ] 有价值条目已标记
- [ ] 至少1条已转入正式知识库
- [ ] 无价值内容已删除
- [ ] _index.md 已更新
```

## 六、工具支持

### 6.1 快速记录脚本

```bash
#!/bin/bash
# quick-note.sh - 快速添加一条记录
# 用法: ./quick-note.sh "记录内容"

INBOX_DIR="d:/Project/personal_knowlegy/知识体系整理/inbox/quick"
TODAY=$(date +%Y-%m-%d)
TIME=$(date +%H:%M)
FILE="$INBOX_DIR/$TODAY.md"

# 如果文件不存在，创建并添加标题
if [ ! -f "$FILE" ]; then
    echo "# $TODAY" > "$FILE"
    echo "" >> "$FILE"
fi

# 追加记录
echo "## $TIME" >> "$FILE"
echo "$1" >> "$FILE"
echo "" >> "$FILE"

echo "已记录到 $FILE"
```

### 6.2 VS Code / Cursor 快捷方式

可以配置快捷键，快速打开今日记录文件：

```json
// keybindings.json
{
    "key": "ctrl+shift+n",
    "command": "workbench.action.files.openFile",
    "args": {
        "path": "${workspaceFolder}/知识体系整理/inbox/quick/${date}.md"
    }
}
```

### 6.3 AI辅助复盘Prompt

```
你是知识整理助手。

以下是我本周的零散记录：
{inbox_content}

请帮我：
1. 识别有沉淀价值的条目（可复用、填补认知缺口、纠正错误认知）
2. 对有价值条目，建议归入哪个领域/层级
3. 尝试用四要素（定义、触发、应用、边界）重写核心条目

输出格式：
## 值得沉淀
| 原记录 | 建议归属 | 四要素重写 |
|--------|----------|------------|

## 可删除
[无价值条目列表]

## 需补充
[信息不完整，需要补充的条目]
```
