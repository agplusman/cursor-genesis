# 分布式知识管理架构

> 来源：04-工程落地模板 讨论延伸
> 核心思想：上层管理索引与关联，叶子节点自治自更新

---

## 一、架构概览

基于「一切皆文件」+ Git 哲学，将知识体系设计为分布式架构：

```
知识图谱上层（管理层）
    │
    ├── 记录所有叶子节点项目的元信息
    ├── 管理知识间的关联关系
    └── 推导变体的依据来源

    │ git submodule / sparse-checkout
    ↓
叶子节点项目（数据层）
    ├── cursor-docs/        # Cursor文档项目
    ├── ai-papers/          # AI论文项目
    ├── ...
    └── 每个项目自包含：
        ├── content/        # 知识内容
        └── scripts/        # 自动更新脚本
```

## 二、上层管理项目结构

```
knowledge-graph/                    # 上层管理项目
├── index/                          # 索引层（L1/L2）
│   ├── domains.yaml               # 领域划分（根节点定义）
│   ├── nodes-registry.yaml        # 所有叶子节点项目的注册表
│   └── relations.yaml             # 跨项目的关联关系
│
├── derivation/                     # 推导层
│   ├── methods/                   # 推导方法（04文档的核心）
│   │   └── variant-derivation.md
│   └── records/                   # 推导记录
│       └── cursor-template-derivation.md
│
├── meta/                           # 元认知层
│   ├── 体系规范.md
│   └── 迭代日志.md
│
└── data/                           # 叶子节点挂载点
    ├── cursor-docs/               # git submodule
    ├── ai-papers/                 # git submodule
    └── ...
```

### 2.1 节点注册表（nodes-registry.yaml）

```yaml
# 所有叶子节点项目的注册表
nodes:
  - id: cursor-docs
    repo: git@github.com:you/cursor-docs.git
    path: data/cursor-docs
    layer: L3
    domain: 工具-Cursor
    description: Cursor官方文档及使用经验
    update_frequency: weekly

  - id: ai-papers
    repo: git@github.com:you/ai-papers.git
    path: data/ai-papers
    layer: L3
    domain: 学术-AI
    description: AI前沿论文收集与笔记
    update_frequency: daily
```

### 2.2 领域划分（domains.yaml）

```yaml
# L1 根节点定义
domains:
  - id: tools
    name: 工具体系
    principle: "工具是认知的延伸"
    children:
      - cursor
      - git
      - obsidian

  - id: academic
    name: 学术探索
    principle: "理论指导实践"
    children:
      - ai
      - math
      - philosophy
```

### 2.3 跨项目关联（relations.yaml）

```yaml
# 跨叶子节点的关联关系
relations:
  - source: cursor-docs/skills/code-review
    target: ai-papers/prompt-engineering
    type: 持久
    reason: "代码审查 skill 的设计借鉴了 prompt 工程原理"

  - source: ai-papers/transformer
    target: cursor-docs/context-window
    type: 持久
    reason: "Cursor 的上下文窗口限制源于 Transformer 架构"
```

## 三、叶子节点项目结构

每个叶子节点是独立的 Git 项目，自包含内容和更新机制：

```
cursor-docs/                        # 一个叶子节点项目
├── content/                        # 知识内容
│   ├── official-docs/             # 官方文档
│   ├── my-notes/                  # 个人笔记
│   └── index.yaml                 # 本项目的索引
│
├── scripts/                        # 自动化脚本
│   ├── sync.sh                    # 同步更新脚本
│   ├── validate.sh                # 内容校验
│   └── config.yaml                # 脚本配置
│
└── README.md                       # 项目说明
```

### 3.1 同步脚本示例（sync.sh）

```bash
#!/bin/bash
# 从 Cursor 官方仓库同步最新文档
cd content/official-docs
git pull origin main

# 触发 AI 摘要更新（可选）
python ../scripts/summarize.py

# 更新索引
python ../scripts/update-index.py
```

### 3.2 本地索引（content/index.yaml）

```yaml
# 本项目内的知识索引
nodes:
  - path: official-docs/rules.md
    title: Rules 设计规范
    keywords: [rules, 规则, 约束]

  - path: my-notes/skill-patterns.md
    title: Skill 设计模式总结
    keywords: [skill, 模式, 最佳实践]
```

## 四、Git 关联机制

### 4.1 使用 Git Submodule（推荐）

```bash
# 在 knowledge-graph 项目中添加叶子节点
git submodule add git@github.com:you/cursor-docs.git data/cursor-docs
git submodule add git@github.com:you/ai-papers.git data/ai-papers

# 查看所有叶子节点
git submodule status
# 输出：
# a1b2c3d data/cursor-docs (v1.2.0)
# e4f5g6h data/ai-papers (v0.3.1)

# 更新所有叶子节点
git submodule update --remote
```

### 4.2 .gitmodules 文件

```ini
[submodule "data/cursor-docs"]
    path = data/cursor-docs
    url = git@github.com:you/cursor-docs.git

[submodule "data/ai-papers"]
    path = data/ai-papers
    url = git@github.com:you/ai-papers.git
```

### 4.3 其他项目按需获取

```bash
# 在具体项目中，只拉取需要的部分
git clone --filter=blob:none --sparse git@github.com:you/cursor-docs.git
cd cursor-docs
git sparse-checkout set content/official-docs/rules
```

## 六、设计原则对照

| 原则 | 如何体现 |
|------|----------|
| 目的驱动 | 上层管理索引和关联，叶子节点专注内容 |
| 用途驱动 | 每个叶子节点可独立使用，也可被其他项目引用 |
| 来源追溯 | nodes-registry 记录每个节点的来源和用途 |
| 动态适配 | 叶子节点自带更新脚本，可按需配置更新频率 |

---

## 七、实施步骤

1. **建立上层项目** `knowledge-graph`，定义 `nodes-registry.yaml` 格式
2. **选一个叶子节点试点**（如 cursor-docs），验证 content + scripts 结构
3. **验证 submodule 工作流**：添加、更新、sparse-checkout 到其他项目
4. **逐步迁移现有知识**到对应的叶子节点项目
