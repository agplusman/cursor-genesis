## 一、追踪源分类体系

### 1.1 五类追踪源

| 类型 | 定位 | 示例 | 获取难度 |
|------|------|------|----------|
| 学术类 | 前沿理论、研究进展 | arXiv、Google Scholar、Nature | 中（有API/RSS） |
| 政策类 | 政府动态、法规文件 | 科技部、工信部、地方政府 | 高（多数无RSS，需爬虫） |
| 技术类 | 工具更新、技术趋势 | Cursor、Claude、GitHub | 低（多有Changelog/RSS） |
| 资讯类 | 行业新闻、热点动态 | X、Hacker News、科技媒体 | 低（有API/RSS） |
| 目标类 | 特定目标相关 | 考研网站、目标院校 | 中（按需配置） |

### 1.2 获取方式优先级

```
1. 官方API（最稳定）
   ↓ 无API时
2. RSS/Atom订阅（次优）
   ↓ 无RSS时
3. 爬虫抓取（兜底，需维护）
   ↓ 反爬严格时
4. 手动检查 + AI辅助摘要
```

## 二、通用处理流程

### 2.1 三阶段流程

```
阶段一：获取（Fetch）
  ├─ 定时任务触发 / 手动触发
  ├─ 调用对应获取器（API/RSS/爬虫）
  └─ 输出：原始内容列表

阶段二：筛选（Filter）
  ├─ 输入：原始内容 + 知识图谱索引
  ├─ AI筛选：相关性判断、优先级排序
  └─ 输出：筛选后的学习建议

阶段三：处理（Process）
  ├─ 人工确认学习建议
  ├─ 深入学习 → 进入经验沉淀/目标驱动模式
  └─ 直接入库 → 进入02入库流程
```

### 3.1 推荐源

| 源 | 覆盖领域 | 获取方式 | 说明 |
|-----|----------|----------|------|
| arXiv | CS、数学、物理、AI | API + RSS | 预印本，最新最快 |

### 3.2 arXiv 配置示例

```yaml
source:
  name: arXiv
  type: academic
  base_url: http://export.arxiv.org/api/query

  - cs.LG          # 机器学习

  max_results: 50          # 每次最多50篇
  sort_by: submittedDate   # 按提交时间排序
  sort_order: descending

  # AI筛选时的额外指令
  focus_keywords:
    - agent
    - LLM
    - prompt engineering
    - knowledge graph
  exclude_keywords:
    - survey only  # 排除纯综述（可选）
```

### 3.3 arXiv API 调用示例

```python
import urllib.request
import feedparser

def fetch_arxiv(categories, max_results=50):
    """获取arXiv最新论文"""
    cat_query = "+OR+".join([f"cat:{cat}" for cat in categories])
    url = f"http://export.arxiv.org/api/query?search_query={cat_query}&start=0&max_results={max_results}&sortBy=submittedDate&sortOrder=descending"

    response = urllib.request.urlopen(url)
    feed = feedparser.parse(response.read())

    papers = []
    for entry in feed.entries:
        papers.append({
            "title": entry.title,
            "authors": [a.name for a in entry.authors],
            "summary": entry.summary,
            "link": entry.link,
            "published": entry.published,
            "categories": [t.term for t in entry.tags]
        })
    return papers
```

### 4.3 通用爬虫框架

```python
import requests
from bs4 import BeautifulSoup
from datetime import datetime

def fetch_policy_page(url, selector, keywords=None):
    """获取政策页面内容"""
    headers = {
        "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
    }

    response = requests.get(url, headers=headers)
    response.encoding = 'utf-8'
    soup = BeautifulSoup(response.text, 'html.parser')

    items = []
    for link in soup.select(selector):
        title = link.get_text(strip=True)
        href = link.get('href', '')

        # 关键词过滤（可选）
        if keywords:
            if not any(kw in title for kw in keywords):
                continue

        items.append({
            "title": title,
            "url": href if href.startswith('http') else url + href,
            "fetch_time": datetime.now().isoformat()
        })

    return items

def fetch_policy_detail(url):
    """获取政策详情页内容"""
    headers = {
        "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
    }

    response = requests.get(url, headers=headers)
    response.encoding = 'utf-8'
    soup = BeautifulSoup(response.text, 'html.parser')

    # 提取正文（选择器需根据实际页面调整）
    content_div = soup.select_one('.article-content, .content, #content')
    content = content_div.get_text(strip=True) if content_div else ""

    return content
```

## 六、目标类追踪源

### 6.1 特点

目标类追踪源是动态的，随目标变化而变化：
- 目标确立时：配置相关追踪源
- 目标完成时：归档或删除追踪源

### 6.2 考研场景示例

```yaml
goal: 计算机考研
status: active
deadline: 2026-12

sources:
  - name: 中国研究生招生信息网
    url: https://yz.chsi.com.cn/

      - 报名通知
      - 政策变化
    frequency: monthly  # 报名季加密

  - name: 目标院校研究生院
    url: [具体院校URL]
    focus:
      - 招生简章
      - 复试线
      - 导师信息
    frequency: monthly

  - name: 408真题/资料站
    url: [具体URL]

      - 新真题
```

### 8.1 主调度脚本

```python
#!/usr/bin/env python3
"""
追踪源主调度脚本
用法：
  python track.py --type academic      # 获取学术类
  python track.py --type policy        # 获取政策类
  python track.py --type tech          # 获取技术类
  python track.py --type all           # 获取全部
  python track.py --manual <url>       # 手动添加URL
"""

import argparse
from pathlib import Path

def main():
    parser = argparse.ArgumentParser(description='知识追踪源获取')
    parser.add_argument('--type', choices=['academic', 'policy', 'tech', 'all'],
                        default='all', help='追踪源类型')
    parser.add_argument('--manual', type=str, help='手动添加URL')
    args = parser.parse_args()

    if args.manual:
        # 手动模式：获取指定URL并AI分析
        manual_fetch(args.manual)
    else:
        # 自动模式：按类型获取
        if args.type in ['academic', 'all']:
            fetch_academic()
        if args.type in ['policy', 'all']:
            fetch_policy()
        if args.type in ['tech', 'all']:
            fetch_tech()

        # 汇总并生成学习建议
        generate_suggestions()

if __name__ == '__main__':
    main()
```
